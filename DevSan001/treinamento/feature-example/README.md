# Exemplo de feature: grok-image-automation-v1

Esta pasta demonstra como organizar uma feature de automação de imagens (Grok).

Conteúdo esperado
- `metadata.yaml` — contexto do experimento
- `logs/` — logs de execução
- `data/` — prompts de teste
- `artifacts/` — imagens geradas

Use `ITEM_TEMPLATE.md` como referência para criar outras features.
